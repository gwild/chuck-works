# Compositions

One markdown file per piece — the living score (the Obsidian-stores
pattern, milton#2409): edit the file for current truth; git history is
the evolution log. These four were exported from milton's score issues
(#2417, #2423, #2431, #2441) where the pattern was first proven during
the 2026-06-06/07 jams; future pieces start here directly.

Score conventions (learned live — see docs/chuck-bringup.md for the
seven engine constraints they depend on):

- **Jam state header**: key, tempo, transport (`/start B/N`), FX state
- **Seat table**: lane → agent slot → voice → register → loop length
- **Form**: sections encoded INTRINSICALLY in phrases (A|B inside one
  phrase; the cycle boundary switches sections — no cue machinery)
- **Voice-leading / arrangement rules** the piece depends on
- Edit the score BEFORE key/tempo changes, then announce (the Am/Cm
  clash of 2026-06-06 is why)
